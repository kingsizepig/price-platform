# 用户价格分享平台 - 产品需求文档

## 1. 产品概述

### 1.1 产品定位
- **UGC 用户价格分享平台**：用户自行录入各类商品价格，经过算法加权平均后展示给所有用户
- **核心价值**：让消费者真实掌握各类商品在不同地区、不同渠道的实际成交价格，避免买贵

### 1.2 产品形态
- **Phase 1**：H5 移动端网页（优先开发）
- **Phase 2**：封装为 App（安卓/iOS），壳+H5 架构，尽可能少用原生组件

### 2.2 分类层级设计
- 三级分类：`一级分类 → 二级分类 → 三级分类（品牌）`
- 支持SKU层级：同一个品牌下不同规格分开价格记录
- 管理员可后台新增/编辑/移动分类

## 4. 用户价格录入

### 4.1 价格数据模型

| 字段 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|--------|------|
| 价格ID | 唯一标识 | 是 | - | |
| 商品ID | 关联 | 是 | - | |
| 录入用户ID | 关联 | 是 | - | |
| 价格 | 数字 | 是 | - | 单位：人民币元 |
| 货币 | 字符串 | 否 | CNY | |
| 成交时间 | 日期 | 否 | 今天 | 用户可修改 |
| 销售渠道 | 枚举 | 否 | 线下门店 | 线上电商/线下门店/二手交易/拍卖等 |
| 所在地 | 省市 | 否 | 用户定位获取 | |
| 品相/成色 | 枚举 | 否 | 全新 | 全新/九成新/七成新/二手/临期 |
| 剩余保质期 | 天数 | 否 | 正常 | 仅食品需要 |
| 促销活动 | 字符串 | 否 | 无 | 如"双11促销"、"店庆打折" |
| 发票/小票 | 图片 | 否 | - | 可选上传凭证 |
| 录入时间 | 时间戳 | 是 | - | |
| 状态 | 枚举 | 是 | 有效 | 有效/已删除 |

### 4.2 录入体验优化

**核心原则**：能默认就默认，让用户**只输价格点提交**就完成

```
用户录入流程：
1. 用户搜索 → 选中商品
2. 页面默认填入：
   - 成交时间 = 今天
   - 所在地 = 用户当前定位（可取消）
   - 销售渠道 = 线下门店
   - 品相 = 全新
3. 用户只需要输入价格 → 点击提交
4. （可选）用户点"高级选项"展开修改其他参数
```

## 5. 用户等级与权重体系

### 5.1 用户积分规则

| 行为 | 积分 | 说明 |
|------|------|------|
| 成功录入一个价格 | +5分 | 审核通过后给分 |
| 录入价格被标记为"准确" | +10分 | 管理员比对后确认准确 |
| 录入价格被标记为"不准确" | -20分 | |
| 账号被举报多次核实 | -50分 | |
| 连续30天无录入 | - | 不扣分，等级权重不变 |

### 5.2 用户等级

| 等级 | 积分要求 | 价格权重 |
|------|---------|----------|
| L1 新手 | 0-99 | 0.5 |
| L2 初级 | 100-499 | 0.8 |
| L3 中级 | 500-1999 | 1.0 | （基准权重） |
| L4 高级 | 2000-9999 | 1.2 |
| L5 认证 | 10000+ 且实名认证 | 1.5 |

### 5.3 权重校准机制

1. **管理员定期录入标杆价格**：每个月管理员录入一批市场公认准确的价格作为基准
2. **用户价格与基准比对**：计算用户录入价格与基准的偏差率
3. **偏差率调整权重**：
   - 偏差率 < 5% → 权重保持
   - 偏差率 5%-15% → 权重 × 0.9
   - 偏差率 > 15% → 权重 × 0.7
4. **恶意录入处理**：连续三次偏差 > 30%，直接降一级

- [ ] 编辑已录入价格的参数（价格、渠道、品相）
- [ ] 删除已录入价格
- [ ] 我的自选商品列表

## 6. 价格计算算法

### 6.1 加权平均算法

```
加权平均价格 = Σ(价格 × 用户权重 × 时间衰减系数) / Σ(用户权重 × 时间衰减系数)
```

**时间衰减系数**（因为价格时效性）：
- 7天内：1.0
- 8-30天：0.8
- 31-90天：0.5
- 91天以上：0.2

> 产品初期要求不高，每月计算一次即可，不需要实时

### 6.2 异常值过滤

- 移除偏离中位数 ±50% 以上的极端价格（排除恶意报价）
- 只保留最近90天内的数据计算当前价格

### 6.3 价格展示

- **当前推荐价格**：加权平均后的结果
- **价格分布**：最低/最高/中位数 展示
- **价格走势**：按月展示平均价格变化曲线
- **价格对比**：按不同地区、不同渠道分别展示平均价格

## 7. 前端H5功能结构

### 7.1 首页

- 搜索框（搜商品）
- 分类导航（按一级分类进入）
- 热门商品（热门搜索/多用户查看）
- 价格涨跌榜（本月涨幅/跌幅前五）

### 7.2 商品详情页

- 商品基本信息（名称、分类、规格、图片）
- **当前推荐价格**（显眼展示）
- 价格分布统计
- 按地区分组价格对比表
- 按渠道分组价格对比表
- 价格走势曲线图（最近6个月）
- [加入自选] 按钮
- [我要录入价格] 按钮

### 7.3 自选列表页

- 用户收藏的商品列表
- 显示每个商品当前价格和上月价格涨跌幅
- 点击进入商品详情
- 支持删除自选

### 7.4 价格录入页

- 商品确认（名称、规格）
- 价格输入框（必填）
- 高级选项（折叠默认收起）：
  - 成交时间
  - 销售渠道
  - 品相
  - 定位所在地
- [提交] 按钮

### 7.5 用户中心

- 等级/积分/权重 展示
- 我的录入历史
- 我的自选
- 编辑个人资料

## 8. UI/UX 设计参考

**风格：股票行情软件 + 京东零售**

- **股票风格**：价格走势用K线/折线图，清晰展示波动，数字加粗清晰
- **京东风格**：商品卡片、图文列表、分类导航简洁易用
- **移动端优先**：所有操作单手可完成，按钮足够大
- **信息密度**：适中，不要太拥挤，价格数字要显眼

## 10. 技术选型建议

- **后端**：Node.js / Python Flask / Go 都可，看你熟悉什么
- **数据库**：MySQL 存结构化数据，Redis 缓存热门商品
- **前端H5**：Vue 3 + Vant UI / Tailwind CSS 移动端组件
- **图表**：ECharts 展示价格走势
- **部署**：先整个轻量云服务器就能跑起来，初期用户量不大足够
